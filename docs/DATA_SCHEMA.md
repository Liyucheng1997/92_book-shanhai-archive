# 《山海异志》数据规范（Codex 摆放契约）

> 每只异兽 = `src/data/creatures/` 下**一个独立 JSON 文件**，文件名 = `<slug>.json`。
> 新增/修改图鉴条目时，**只动这个目录**，不要回去改 `App.jsx`。前端通过 `src/data/index.js` 自动收集、排序。
>
> 违反本规范（字段缺失、分类乱填、排序乱放）会导致目录、地图、筛选错乱。生成前先读本文件。

---

## 1. 文件与命名

- 路径：`src/data/creatures/<slug>.json`
- `slug`：小写拉丁拼音，无声调、无空格，与图片文件名一致。例：`xingxing`、`jiuweihu`、`zhulong`。
- 图片：`public/images/creatures/creature_<slug>_v<NN>.webp`，`image` 字段填 `"/images/creatures/creature_<slug>_v03.webp"`。
- 一个 slug 只对应一个文件；改图只改 `image` 的版本号，不新建文件。

---

## 2. 必填字段

| 字段 | 类型 | 说明 |
|---|---|---|
| `slug` | string | 唯一 id，见上 |
| `name` | string | 中文名，如「狌狌」 |
| `alias` | string | 副标题/异名，如「又名 烛龙」 |
| `pinyin` | string | 大写拼音，多音用 `·` 分隔 |
| `type` | enum | **分类**，见 §3 |
| `tags` | string[] | **属性标签**，见 §4 |
| `volume` | int 1–18 | **卷序**，见 §5，用于排序 |
| `book` | string | 篇名，如「南山经」 |
| `scroll` | string | 卷号汉字，如「卷一」 |
| `seq` | int | 该卷内出场顺序，用于排序（同卷按原文先后递增） |
| `plate` | string | 图鉴编号汉字（壹贰叁…），按全局收录顺序，可后补 |
| `mark` | string | 印章用单字，通常取名字首字 |
| `location` | string | 古籍地点，如「招摇之山」 |
| `region` | string | 所属山系/区域，如「南山首经」「北方山海」 |
| `mapPos` | {x,y} | 示意地图坐标，**0–1 小数**，见 §6 |
| `geoConfidence` | enum | `none`\|`low`\|`medium`，现代地理对应可信度（几乎都填 `low`，神话天神填 `none`） |
| `image` | string | 见 §1 |
| `imageAlt` | string | 语义化 alt |
| `imageTag` | string | 图卡副标，3 个特征用 ` · ` 连，如「人面 · 蛇身 · 赤色」 |
| `quote` | string | **原文核录**，必须查证，不加书名号外层引号 |
| `summary` | string | 80–140 字白话导读 |
| `facts` | Fact[] | 4 条特征卡，见 §7 |
| `sourceNote` | string | 出处与考据说明（差异、注家分歧、艺术推定声明） |

可选字段：`imageClass`（特殊比例图用，一般留空）、`references`（出处列表，见 §8）。

## 3. `type` 分类（六类，决定筛选归属）

| 值 | 中文 | 范围 |
|---|---|---|
| `deity` | 神 | 天神、方位神、人格神（烛阴、句芒、陆吾） |
| `beast` | 兽 | 走兽（狌狌、鹿蜀、穷奇） |
| `bird` | 禽 | 鸟类（精卫、毕方、颙） |
| `fish` | 鳞 | 鱼及水生（赤鱬、文鳐鱼、何罗鱼） |
| `serpent` | 蛇虫 | 蛇、虫、龙蛇（肥遗、相柳、应龙） |
| `figure` | 异人 | 人或半人形（夸父、刑天、贰负） |

混合体按**主体/最强辨识部位**归类；拿不准时以原文起首「有兽/有鸟/有鱼/有蛇」为准。

## 4. `tags` 属性标签（可多选，决定语义徽章与筛选）

固定词表，**只从下列取值**，不要自创：

`食人`、`御凶`、`旱兆`、`水兆`、`火兆`、`疫兆`、`丰年`、`宜子孙`、`不畏`、`毒`、`神能`、`司时`、`填海`、`逐日`。

无合适标签可留空数组 `[]`。

## 6. `mapPos` 示意地图坐标

地图是**相对方位示意**，不是真实地理。画布按罗盘布局：

```
        北 / 海外·大荒北 (y≈0.10)
西 (x≈0.15)      中山 (中心)      东 (x≈0.85)
        南 / 海外·大荒南 (y≈0.90)
```

- `x`、`y` 取 0–1 小数，按该异兽所属山系大致落位，同一山系内沿链条错开，避免重叠（间隔 ≥0.06）。
- 五藏山经：南→底部、西→左、北→顶部、东→右、中→中心带。
- 海外/海内/大荒：放到对应方向更外圈（更靠边缘）。

## 7. `facts` 特征卡（每只恰好 4 条）

```json
{ "label": "形貌", "value": "如禺而白耳", "basis": "text" }
```

`basis` 枚举，**决定前端可信度徽章**：

| basis | 徽章 | 含义 |
|---|---|---|
| `text` | 原文 | 原文明示 |
| `inferred` | 推定 | 为补全解剖/造型所作的艺术推定 |
| `note` | 注 | 注家说法、读音、传世异说 |

至少保留 2 条 `text`；凡 `inferred` 必须在 `sourceNote` 里说明推定理由。

---

## 8. `references` 出处（可选，做深考据用）

```json
"references": [
  { "book": "海外北经", "text": "钟山之神，名曰烛阴……" },
  { "book": "大荒北经", "text": "西北海之外……有神，人面蛇身而赤……" }
]
```

多篇出处有差异时尽量都列，体现「保留古籍差异、不把推测当定论」的项目立场。

---

## 9. 新增条目检查清单

1. 文件名 = `<slug>.json`，slug 与图片名一致。
2. `type` 取自 §3、`tags` 取自 §4、`volume` 取自 §5。
3. `seq` 在同卷内按原文顺序排，避免与已有条目重号。
4. `quote` 已核对古籍原文。
5. `facts` 恰好 4 条且 `basis` 正确。
6. `mapPos` 不与同区域条目重叠。
7. 不改 `App.jsx`、不改 `index.js`。
